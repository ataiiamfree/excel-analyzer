# ChatExcel v0.9 Max 视频脚本

目标成片：90–120 秒<br>
素材原则：真实浏览器、真实 Excel、真实分析结果；不使用 AI 生成视频，不伪造输出。<br>
固定问题与标准答案：见 `v0.9-demo-runbook.md` 和 `v0.9-demo-expected-results.md`。

## 录制策略

先完整录一轮真实操作，再剪出 90–120 秒版本。2026-07-19 至 2026-07-20 的实测首问为 48.2–83.0 秒，追问为 54.1–114.9 秒，因此不剪辑的原始录屏很可能超过 3 分钟。成片可对等待段做等比加速或剪短，但必须保留“提交问题 → 正在分析 → 本轮真实结果”的顺序。

## 录屏操作清单

1. 运行 `python scripts/demo_preflight.py`，必须 10/10。
2. 按 `v0.9-demo-checklist.md` 清理通知、窗口和下载目录。
3. 使用 1920×1080、100% 缩放录主版本；1440×900 作为备份尺寸。
4. 主问题和追问只发送一次，不临场改写。
5. 分析中不展开“分析过程”或“技术详情”。
6. 首问结果依次展示：结论先行 → 图表 → CSV → Excel → 下载入口。
7. 追问结果打开 `销售额TOP2对比.csv` 或同类型的 2 行 CSV。
8. 最后刷新，确认历史恢复后停止录制。

## 剪辑边界

允许：裁掉静默等待、等比加速真实等待、放大页面已有区域、加旁白与字幕。

不允许：重排操作顺序、用另一轮结果伪装为本轮、改页面数字、隐藏失败后声称实时成功、使用 AI 生成的产品运行画面。

## 网络异常备用段

旁白固定为：

> 网络暂时不可用，我切到刚才已经完成并核对过的同一分析，继续展示报告和产物。

随后从侧栏打开已完成会话。保留这句话比假装实时成功更可信。
