# ChatExcel v0.9 Max 演示检查单

适用版本：v0.9 Max（0.9.2）

这是一张只读、可重复执行的演示检查单。它不会删除会话、修改 `.env`、启动或停止服务，也不会输出 API key。

## 提前 15 分钟

- [ ] 开启系统勿扰模式，关闭邮件、IM、日历和系统通知。
- [ ] 关闭终端、密钥、私人文件和无关浏览器标签页。
- [ ] 浏览器缩放为 100%；录制首选 1920×1080，现场演示至少使用 1440×900。
- [ ] 使用固定工作簿 `docs/test_datasets/simple/01_门店月度销售.xlsx`。
- [ ] 主问题和追问从 `v0.9-demo-runbook.md` 复制，不临场改写。
- [ ] 先生成 production build，再使用稳定单端口服务：

  ```bash
  make web-build
  make run
  ```

- [ ] 服务启动后运行：

  ```bash
  python scripts/demo_preflight.py
  ```

- [ ] 预检结果应为 10/10；任何红叉都先处理，不带病录制。
- [ ] 下载目录可写且容易找到；不要在镜头中打开包含私人文件的下载列表。
- [ ] 保留一个已真实跑完、与标准答案核对一致的备用会话。

## 开始前 2 分钟

- [ ] 首页只出现 ChatExcel、`本地工作区` 和 `v0.9 Max`，没有个人姓名。
- [ ] 首页标题、上传区、输入框、示例问题和开始按钮在一屏内。
- [ ] 原始工作簿可打开，包含 `月度销售` 表和 30 行数据。
- [ ] 手边保留 `v0.9-demo-expected-results.md`，重点核对 753.3、739.7、664.9、664.8、566.7。
- [ ] 明确本次是本地单用户 MVP 演示，不使用“生产级”“100% 准确”等表述。

## 演示过程中

- [ ] 主问题只发送一次；分析中不刷新、不重复点击发送。
- [ ] “分析过程”默认收起；主画面不展开 token、stdout、脚本路径或内部推理。
- [ ] 首问必须出现完整 5 家门店排名、图表、CSV 和 Excel。
- [ ] 追问必须出现杭州 88.4、北京 74.8，以及仅 2 行的 CSV。
- [ ] 下载只需证明入口有效；正式镜头不打开可能保留原始浮点精度的 CSV 文本。
- [ ] 最后刷新一次，确认消息、报告和产物恢复。

## 异常切换话术

## 结束后

- [ ] 视频/截图中的数值逐项匹配标准答案。
- [ ] 没有通知、密钥、绝对路径、个人会话或终端入镜。
- [ ] PPT、视频旁白和交付文档都使用 `ChatExcel v0.9 Max（0.9.2）`。
- [ ] 如本机无法可靠录屏，交付 runbook、视频脚本、截图和视觉 QA，由演示者按脚本现场跑。
