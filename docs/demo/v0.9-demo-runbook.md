# ChatExcel v0.9 Max 演示 Runbook

适用版本：v0.9 Max（0.9.2）

演示原则：只展示真实浏览器、真实 Excel 和真实分析运行；不使用 AI 生成视频，不伪造模型输出。录制困难时，本文件与 QA 证据可以直接作为现场演示手册。

## 一句话故事

把一张需要人工分组汇总的门店月度销售表交给 ChatExcel，用一句自然语言得到可核对的排名、图表和可下载表格，并能继续追问而无需重新上传。

## 固定演示材料

- Excel：`docs/test_datasets/simple/01_门店月度销售.xlsx`
- 标准答案：`docs/demo/v0.9-demo-expected-results.md`
- 主问题：

  > 分析 2025 年上半年各门店总销售额，给出从高到低的完整排名，生成一张横向条形排名图，并导出包含排名、门店和总销售额的 CSV 与 Excel。

- 追问：

  > 只保留销售额最高的两个门店，并说明它们分别比第三名高多少，输出 CSV。

录制和彩排时保持以上文字不变，避免同义改写导致计划和产物类型随机变化。

## 录制前准备

1. 开启系统“勿扰模式”，关闭邮件、IM 和日历通知。
2. 关闭包含 API key、终端绝对路径或私人内容的窗口。
3. 使用 production build 的单端口模式：

   ```bash
   make web-build
   make run
   ```

4. 浏览器使用 100% 缩放；首选 1920×1080 录制，至少验证 1440×900。
5. 下载目录保持干净，确保 CSV/XLSX 下载后容易定位。
6. 运行 `python scripts/demo_preflight.py`，预期 10/10 通过。
7. 先完成一次不录制的彩排并核对标准答案。

逐项检查见 `docs/demo/v0.9-demo-checklist.md`。

## 90–120 秒成片版

| 时间 | 画面与操作 | 建议旁白 | 验收点 |
|---|---|---|---|
| 0–10s | ChatExcel 首页全景 | “面对一张 Excel，不需要先写公式或透视表，直接描述想知道的问题。” | 品牌、价值文案、上传区同屏 |
| 10–25s | 选择固定 Excel，粘贴主问题，点击开始分析 | “这里是一张五家门店的上半年月度销售表。我需要完整排名、图表和可下载结果。” | 文件名和问题清晰可见 |
| 25–50s | 展示计划和实时进度 | “ChatExcel 会先理解表格结构，再执行计算、校验结果并生成产物。” | 不展示 token、脚本路径或 stdout；等待段可等比加速 |
| 50–75s | 滚动到最终答案，停留在完整排名 | “杭州西湖店以 753.3 万元排名第一，北京旗舰店为 739.7 万元。” | 主数字与标准答案一致 |
| 75–90s | 打开排名图和表格预览，点击下载 | “结论不仅能看，还会生成图表、CSV 和 Excel，方便复核与继续使用。” | 图表顺序正确；表格 5 行；下载可触发 |
| 90–108s | 发送固定追问，展示 Top 2 与第三名差值 | “同一会话可以直接追问，不需要重新解释字段或重新上传。” | Top 2、88.4 和 74.8 正确 |
| 108–120s | 刷新页面，展示历史与产物恢复 | “结果和产物会保留。v0.9 Max 面向本地单用户演示和自用。” | 刷新后内容完整恢复 |

录制可以剪掉网络等待，但不得剪接出不存在的操作顺序，也不得用预制结果冒充本轮实时输出。

## 3 分钟现场演示版

1. **20 秒定位**：说明“上传 Excel、自然语言分析、得到报告和产物”，同时指出这是本地单用户 MVP，不宣称企业级生产可用。
2. **35 秒输入**：打开原始工作簿简单展示列名，然后回到首页上传，粘贴固定主问题。
3. **45–90 秒分析**：让观众看到计划和步骤状态；解释系统会解析、计算和校验，不展开内部推理细节。
4. **45 秒结果**：核对杭州 753.3、北京 739.7；打开图表、表格和下载入口。
5. **25 秒追问**：发送固定 Top 2 追问，展示上下文连续性。
6. **15 秒稳定性**：刷新页面，展示消息和产物恢复；最后说明复杂表格仍存在 6 个已知准确性限制。

如果实时运行超过 90 秒，继续解释执行计划和产品边界，不频繁点击、不刷新、不重复发送问题。真实彩排中首问耗时 48.2–83.0 秒，追问耗时 54.1–114.9 秒；3 分钟现场版需要为追问准备已完成备用会话。

## 备用会话规则

在正式录制或现场演示前，保留一个使用同一工作簿、同一主问题且已经人工核对通过的会话。

仅在网络或模型服务不可用时使用：

1. 明确说“网络暂时不可用，我切到刚才已经完成并核对过的同一分析”。
2. 从侧栏切换到备用会话，展示报告、图表、表格和下载。
3. 不声称备用会话是刚刚实时生成的。
4. 不修改网页、数据库或录屏来隐藏故障。

## 失败时停止条件

遇到以下任一情况，停止正式录制，修复或重新彩排：

- 主排名、数值或单位错误。
- 未生成要求的图表、CSV 或 Excel。
- 页面出现个人信息、API key、本地绝对路径、token、stdout 或明显英文调试标签。
- 表格出现明显浮点尾差，或关键列被裁切。
- WebSocket 中断后无法通过一次手动重连恢复。

说明：断线或标签页关闭会把当轮任务明确标记为“已取消”。重新打开历史会话后可以继续提问；不得声称取消的任务仍在后台完成。

## 录制完成后的快速复核

- 视频中的门店名称和数值与标准答案逐项一致。
- 等待加速或剪辑没有改变真实操作顺序。
- 没有系统通知、私人会话名、终端、密钥或绝对路径入镜。
- 视频、截图、PPT 和 release note 均使用 `ChatExcel v0.9 Max（0.9.2）`。
