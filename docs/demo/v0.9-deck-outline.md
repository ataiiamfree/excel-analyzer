# ChatExcel v0.9 Max PPT 大纲

建议页数：8 页<br>
统一口径：ChatExcel v0.9 Max（0.9.2）<br>
定位：本地单用户 MVP；不宣称企业级生产可用或 100% 准确。

## 1. 痛点：Excel 分析仍然消耗大量手工时间

**页面主句**：数据在 Excel 里，但结论、复核和分享仍要反复手工处理。

**三点内容**：

- 字段和表头不总是规整，先整理再分析。
- 汇总、排序、画图和导出分散在多步操作里。
- 结果难复核，追问时经常需要重做一遍。

**视觉**：固定演示工作簿的局部截图或简单三段痛点图，不放未经授权的客户数据。

## 2. 产品定位：上传 Excel，用一句话得到可复核结果

**页面主句**：上传 Excel，用自然语言获得分析结论、图表和可下载表格。

**能力标签**：自动理解表格 / 可追问分析 / 结果可下载。

**视觉**：`screenshots/01-home-clean-1440x900.png`。

## 3. 用户流程：从文件到结论只保留四步

**流程**：上传 → 提问 → 自动分析 → 报告与产物。

**讲解重点**：

- 用户描述业务目标，不需要写代码。
- 页面保留可解释的执行步骤，但默认隐藏技术噪声。
- 结果既可阅读，也可继续使用。

**视觉**：四步横向流程，配合 `screenshots/06-analysis-in-progress-1440x900.png`。

## 4. 产品实录：分析过程清晰，但不打扰业务阅读

**页面主句**：状态、计划和进度可见；内部推理与脚本细节默认收起。

**证据**：分析过程收起、中文任务标签、停止按钮、无个人信息/token/stdout。

**视觉**：`screenshots/06-analysis-in-progress-1440x900.png`，裁切时保留浏览器主画面与产物面板。

## 5. 结果证据：排名、图表和表格相互校验

**页面主句**：杭州西湖店 753.3 万元排名第一，五家门店结果与源表独立核算一致。

**关键数字**：753.3 / 739.7 / 664.9 / 664.8 / 566.7。

**视觉**：主图使用 `screenshots/04-summary-and-artifacts-1920x1080.png`；补充 `screenshots/02-summary-1440x900.png` 的结论先行区域。

**证据来源**：`v0.9-demo-expected-results.md`，不是模型输出反推。

## 6. 连续工作：同一会话继续追问并恢复历史

**页面主句**：不重新上传即可追问；刷新后消息和产物仍在。

**关键数字**：杭州比第三名高 88.4 万元，北京比第三名高 74.8 万元。

**视觉**：`screenshots/05-follow-up-recovery-1440x900.png`。

## 7. 稳定性证据：流程可跑通，失败可解释、可恢复

**页面内容**：

- Python 全量测试与前端 production build 通过（最终数字以 QA 报告为准）。
- 固定首问连续两次正确，固定追问连续两次正确。
- 真实断线被标记为取消；重新打开历史后可继续追问。
- 错误格式、损坏文件、空问题、取消和运行中删除都有明确边界。
- 当前 P0 = 0。

**视觉**：小型证据表，不堆终端截图。引用 `docs/v0.9-max-qa-report.md` 与 `v0.9-demo-visual-qa.md`。

## 8. 边界与下一步：MVP 已可交付，能力仍需迭代

**当前边界**：

- 本地/内网单用户，不包含鉴权、多租户、监控和生产部署。
- SheetBench complex-qa 为 15/21；6 个剩余 case 作为已知准确性限制。
- LLM 运行有时延和随机性；实时演示需准备已核对的备用会话。
- 个别生成 CSV 的原始文本可能保留浮点精度尾差，UI 预览已整洁显示。

**下一步**：优先提升复杂表格语义、任务恢复、artifact/ workspace GC 和生产化安全边界。

**收尾句**：v0.9 Max 的目标不是“企业级生产系统”，而是把每条核心流程稳定跑通，并提供可以复核和交付的结果。

## 版式建议

- 每页只保留一个结论句和最多三个证据点。
- 产品截图使用原始比例，不把不同轮次画面拼成一张“伪流程”。
- 颜色沿用产品的暖白背景、砖红强调色、深绿成功色。
- 正文字号不低于 20pt；关键数字使用 32–40pt。
- 页脚统一写 `ChatExcel v0.9 Max · 0.9.2`。

## 生成 PPT 前的确认点

大纲确认后再创建 `ChatExcel-v0.9-Max-Demo.pptx`。生成时只使用本目录的真实截图和当前 QA/release 文档，不新增未经证实的用户数、节省时间比例或准确率。
