# Excel Analyzer v0.9 MVP 交付说明

交付版本：`v0.9.2-mvp`（产品展示名：v0.9 Max）<br>
最终验收日期：2026-07-20

## 交付结论

v0.9.2 已达到本地单用户 MVP 的可交付候选状态：核心分析链路、会话恢复、错误边界、结果产物和稳定启动方式均可用。真实浏览器验收中发现的 Agent 越权 Git 操作、开发热重载中断分析和空白问题守卫已经修复。此版本不以 SheetBench 全通过为目标，也不宣称具备生产环境的鉴权、多租户和运维能力。

## 已交付能力

- 上传并校验 `.xlsx` / `.xlsm`，拒绝超限、损坏、加密或错误格式文件。
- Excel ingest、表头识别、normalized table、数据画像、规划、Python 执行、结果校验和报告生成。
- 普通平表、多 Sheet、重复表头、多层列头、行分组上下文和公式表头的通用解析增强。
- React Web 会话：首页上传、分析进度、追问、取消、断线恢复、历史会话、替换工作簿和产物预览。
- 回答操作：重新生成、复制、导出；取消和失败步骤会持久化并正确显示。
- SQLite 会话与消息持久化。服务重启后历史会话仍可读取。

## 已知限制

最终 21 例中的 6 个失败均为答案准确性问题，不是服务崩溃：

| Case | 当前限制 |
|---|---|
| 104 | 英文月份解析失败后仍给出结论，缺少季度复合增长率数值 |
| 101 | 同一序号分布在多个分组时，只返回了前两个匹配项 |
| 126 | 多层 `Direct Cost` 列之间的业务口径选择不稳定 |
| 2292 | 明细行和 `of which` 子行同时求和，产生重复统计 |
| 1135 | 行分组上下文已保留，但整批运行中重复汇总为 48 而非 24 |
| 515 | 精确汇总行在单例中可命中，整批运行时模型仍可能返回 `Not Found` |

普通 QA 的预算案例正确给出销售部、预算、实际和执行率，但导出表没有单独输出 `超支金额=23`，因此严格产物断言失败。这属于输出完整性问题，不是主结论错误。

其他边界：

- 当前只支持 `.xlsx` / `.xlsm`，不直接支持旧版 `.xls`、CSV 或在线表格。
- 分析不会原地编辑用户工作簿；需要结构化结果时会新建 Excel/CSV 等产物。
- LLM 生成代码存在一定随机性，同一复杂问题可能得到不同结果。
- 活跃任务不会跨服务关闭继续运行；关闭服务前应等待完成或先取消。
- 当前面向本地/内网单用户，不包含鉴权、多租户、审计、监控和生产部署。
- 个别 Agent 生成 CSV 的原始文本可能保留 Python 浮点精度尾差；Web UI 预览已格式化为业务小数位。

## 运行方式

开发时分别运行后端 `make run` 和前端 `make web-dev`，访问 `http://127.0.0.1:5173`。停止时在两个终端分别按 `Ctrl+C`，重启时重新执行相同命令。完整安装和单端口运行方式见仓库根目录 [README](../README.md)。
