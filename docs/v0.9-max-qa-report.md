## 验收发现及修复

### P0-1：数据分析 Agent 越权执行 Git

首次真实运行中，Pi 默认读取仓库 `CLAUDE.md`，把“每次有改动后都需要 Git 提交”的编码规则错误套用到数据分析任务，执行了 `git add -A && git commit`。该提交已用 mixed reset 安全撤销，所有原文件完整保留为未跟踪状态。

修复：

- Pi sidecar 强制加入 `--no-context-files`，隔离 `CLAUDE.md` / `AGENTS.md`。
- 默认只启用调用 typed tool service 所需的 `bash`。
- Pi 子进程强制 `GIT_DIR=/dev/null`，阻断仓库发现与写入。
- runtime prompt 明确禁止 Git、修改源码/文档、安装依赖和提交。
- 新增回归测试验证命令行、环境和 prompt 防线。

复测：同会话真实追问成功，报告/CSV 正常，仓库 `HEAD` 未变化。

### P0-2：`make run --reload` 被运行时脚本误触发

原 `make run` 监控整个仓库。Agent 在 `workspace/<session>/scripts/` 生成 Python 脚本时会触发 Uvicorn reload，导致 WebSocket 中断。

修复：默认 `make run` 改为稳定无 reload；新增 `make run-dev`，只监控 `app/`。

### P1-1：空白问题仍可点击开始

首页只检查文件，未对 query 执行 `trim()`。已在提交入口、mutation 和按钮 disabled 条件中统一加入空白守卫；最新生产构建通过真实浏览器复测。
